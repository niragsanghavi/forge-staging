User Identity Migration (S1)
Backfill users/{userId}
Creates one users doc per person across all groups (exact-uid merge only — never by name) and writes userId back onto rosters. Idempotent: safe to dry-run repeatedly, safe to re-run live. Preview first — Run Migration stays locked until a dry run has completed.
Stats Backfill (S1b)
Recompute users/{userId}.stats from logs
Recomputes totalWorkouts, longest & current streak, lastLoggedDay, seasonsPlayed (distinct group-months with a log), and monthsLogged (distinct CALENDAR months — the same month in two groups counts once) from real logs — per canonical userId (mergedInto-resolved). Badges are left untouched. Idempotent: overwrite, not increment — safe to re-run. Preview first — Run stays locked until a dry run completes.
